# 핵심 설계 신념 (Core Beliefs)

> 에이전트가 판단에 맞닥뜨릴 때 기준이 되는 **변하지 않는 원칙들**.
> 구체적 규칙(`AGENT.md`)이 답하지 못하는 엣지 케이스를 이 신념으로 판단한다.

## 1. 단순함이 영리함을 이긴다
- 비슷한 세 줄이 섣부른 추상화보다 낫다.
- 가상의 미래 요구를 위해 설계하지 않는다.

## 2. 환경이 코드보다 중요하다 (하네스 우선)
- 같은 실수가 반복되면 그 실수를 **개별 수정**하지 말고, 그것을 막는 **규칙/검사를 환경에 추가**한다.
- 모델/접근을 바꾸기 전에 하네스(지침·검사·피드백)를 먼저 점검한다.

## 3. 피드백 루프는 빠르고 읽을 수 있어야 한다
- 검증 실패는 에이전트가 스스로 고칠 수 있도록 원인과 수정법을 함께 알려준다.
- "통과/실패"만 반환하는 검사는 가치가 절반이다.

## 4. 엔트로피는 방치하면 증가한다
- 코드 표류·중복·죽은 코드는 적극적으로 청소한다(garbage collection).
- 단, 작업 범위를 벗어난 청소는 별도 작업으로 분리한다.

## 5. 되돌릴 수 없는 행동은 멈춰서 확인한다
- 파괴적·외부 가시적·하드복구 작업은 실행 전 사용자 확인.
