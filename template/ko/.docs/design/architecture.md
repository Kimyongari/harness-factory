# 아키텍처 경계 (Architecture Boundaries)

> 에이전트가 코드 표류(drift)를 일으키지 않도록 하는 **불변의 구조 규칙**.
> 이 규칙은 `.scripts/check-boundaries.sh`로 기계적으로 강제된다(설명만 두지 않는다).

## 의존성 방향 (단방향)
의존성은 한 방향으로만 흐른다. 역방향 import는 금지하며 린터가 차단한다.

```
{{FILL:dev.architecture_layers}}
(왼쪽이 오른쪽을 모른다. 화살표 방향으로만 의존한다.)
```

- 맨 왼쪽 레이어는 아무것도 import 하지 않는다(순수 타입/도메인 모델).
- 맨 오른쪽 레이어는 모든 하위 레이어를 쓸 수 있지만, 어떤 하위 레이어도 그것을 import 할 수 없다.
- 위반 시: 빌드 실패 + 위반 경로를 에이전트가 읽을 수 있는 형태로 출력.

> `{{FILL:dev.architecture_layers}}`가 비어 있으면 이 검사는 건너뛴다(과잉 제약 방지).
> 레이어가 없다면 위 블록을 비워 두면 검사가 자동으로 생략된다. 경계를 도입하려면 여기에 `왼쪽 -> 오른쪽` 순서로 적는다.

## 경계 규칙
- 레이어 간 통신은 인터페이스/포트를 통해서만. 구체 구현을 가로질러 참조하지 않는다.
- 순환 의존성 금지.
- 외부 I/O(DB, 네트워크, 파일)는 `Repo`/`Runtime` 레이어에 격리한다.

## 강제 방식 (Mechanical Enforcement)
- 규칙을 문서로만 두면 에이전트가 어긴다. 반드시 **자동 검사 + 명확한 에러 메시지**로 강제한다.
- 검사 위치: `.scripts/check-boundaries.sh` (verify 파이프라인에 포함).
- 에러는 "무엇을, 어디서, 왜" 위반했고 "어떻게 고치는지"를 한 번에 알려준다.
